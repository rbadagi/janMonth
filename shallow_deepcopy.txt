In Python, **copy** and **deep copy** describe two different ways of duplicating objects—especially important when working with **mutable objects** like lists, dictionaries, or custom objects.

***

## 1. Shallow Copy (`copy.copy()` or `.copy()`)

A **shallow copy** creates a new outer object, but **does not copy the nested (inner) objects**.  
Instead, it copies **references** to those inner objects.

### Key idea

*   Outer container → **new**
*   Inner objects → **shared**

### Example

```python
import copy

original = [[1, 2], [3, 4]]
shallow = copy.copy(original)

shallow[0].append(99)

print(original)   # [[1, 2, 99], [3, 4]]
print(shallow)    # [[1, 2, 99], [3, 4]]
```

✅ `original` and `shallow` are separate lists  
❌ But their inner lists are the **same objects**

### When shallow copy is used

*   Simple objects
*   When nested objects should be shared
*   Better performance and lower memory usage

***

## 2. Deep Copy (`copy.deepcopy()`)

A **deep copy** creates a new object **and recursively copies all nested objects**.

### Key idea

*   Outer container → **new**
*   Inner objects → **also new**

### Example

```python
import copy

original = [[1, 2], [3, 4]]
deep = copy.deepcopy(original)

deep[0].append(99)

print(original)  # [[1, 2], [3, 4]]
print(deep)      # [[1, 2, 99], [3, 4]]
```

✅ Changes to `deep` do **not** affect `original`

### When deep copy is used

*   Complex nested structures
*   When full independence is required
*   Avoiding unintended side effects

***

## 3. Visual Comparison

| Aspect            | Shallow Copy | Deep Copy    |
| ----------------- | ------------ | ------------ |
| Outer object      | New          | New          |
| Inner objects     | Shared       | Fully copied |
| Safer for changes | ❌ No         | ✅ Yes        |
| Performance       | Faster       | Slower       |
| Memory usage      | Less         | More         |

***

## 4. Common Ways to Create Copies

```python
import copy

copy.copy(obj)        # Shallow copy
copy.deepcopy(obj)    # Deep copy

# Shallow copy alternatives
list_copy = my_list[:]
dict_copy = my_dict.copy()
```

⚠️ These alternatives still perform **shallow copies**.

***

## 5. Quick Rule of Thumb

*   ✅ **Use shallow copy** if the object contains **only immutable items** (ints, strings, tuples)
*   ✅ **Use deep copy** if the object contains **mutable nested elements**

Here are **clear, practical examples using dictionaries** to show the difference between **shallow copy** and **deep copy** in Python.

***

## Example Setup

We’ll use a dictionary that contains **nested mutable objects** (lists and dictionaries):

```python
import copy

original = {
    "name": "Alice",
    "skills": ["Python", "SQL"],
    "address": {
        "city": "Bangalore",
        "pincode": 560001
    }
}
```

***

## 1. Shallow Copy with Dictionaries

### Creating a shallow copy

```python
shallow = copy.copy(original)
# or
shallow = original.copy()
```

### Modifying nested data

```python
shallow["skills"].append("Snowflake")
shallow["address"]["city"] = "Mysore"
```

### Output

```python
print(original)
print(shallow)
```

✅ Output:

```text
{
 'name': 'Alice',
 'skills': ['Python', 'SQL', 'Snowflake'],
 'address': {'city': 'Mysore', 'pincode': 560001}
}
```

### Why this happens

*   The **outer dictionary** is copied
*   The **inner list and inner dictionary are shared**
*   Changing nested values affects both

***

## 2. Deep Copy with Dictionaries

### Creating a deep copy

```python
deep = copy.deepcopy(original)
```

### Modifying nested data

```python
deep["skills"].append("Databricks")
deep["address"]["city"] = "Hubli"
```

### Output

```python
print(original)
print(deep)
```

✅ Output:

```text
Original:
{
 'name': 'Alice',
 'skills': ['Python', 'SQL'],
 'address': {'city': 'Bangalore', 'pincode': 560001}
}

Deep Copy:
{
 'name': 'Alice',
 'skills': ['Python', 'SQL', 'Databricks'],
 'address': {'city': 'Hubli', 'pincode': 560001}
}
```

### Why this works

*   Every nested object is **recursively copied**
*   Changes in `deep` do **not** affect `original`

***

## 3. Special Case: Updating Top-Level Keys

Even with a **shallow copy**, updating a **top-level immutable value** does NOT affect the original.

```python
shallow = original.copy()
shallow["name"] = "Bob"

print(original["name"])  # Alice
print(shallow["name"])   # Bob
```

✅ Because strings are immutable, a new reference is created.

***

## 4. Memory Reference Proof (Interview Favorite)

```python
print(id(original["skills"]) == id(shallow["skills"]))  # True
print(id(original["skills"]) == id(deep["skills"]))     # False
```

✔ Shallow copy → same inner object  
✔ Deep copy → different inner object

***

## 5. Summary Table (Dictionaries)

| Operation                        | Shallow Copy | Deep Copy |
| -------------------------------- | ------------ | --------- |
| `dict.copy()`                    | ✅            | ❌         |
| `copy.copy()`                    | ✅            | ❌         |
| `copy.deepcopy()`                | ❌            | ✅         |
| Nested mutation affects original | ✅ Yes        | ❌ No      |

***

## 6. Rule of Thumb (Very Important)

*   ✅ Use **shallow copy** if:
    *   Dictionary contains only **immutable values**
    *   You intentionally want shared nested data
*   ✅ Use **deep copy** if:
    *   Dictionary has **lists, dicts, sets, or objects**
    *   You want full independence

Here’s a **practical, performance-focused comparison** of **shallow copy vs deep copy** in Python, with time/space cost, Big‑O intuition, and real-world guidance.

***

## High-Level Performance Comparison

| Aspect                | Shallow Copy              | Deep Copy                      |
| --------------------- | ------------------------- | ------------------------------ |
| Time complexity       | **O(n)** (top-level only) | **O(N)** (entire object graph) |
| Memory usage          | **Low**                   | **High**                       |
| Speed                 | **Fast**                  | **Slower**                     |
| Copies nested objects | ❌ No (shared references)  | ✅ Yes (recursive)              |
| Risk of side effects  | ❌ Higher                  | ✅ Lower                        |

> `n` = number of top-level references  
> `N` = total number of objects including nested ones

***

## Why Shallow Copy Is Faster

### What it does internally

*   Allocates a **new container**
*   Copies **references** to existing inner objects
*   Does *not* traverse nested structures

```python
import copy

shallow = copy.copy(big_dict)
```

### Performance implications

✅ Minimal allocation  
✅ No recursion  
✅ Constant-time operations per key

👉 Best for **large objects with shared data**

***

## Why Deep Copy Is Slower

### What it does internally

*   Allocates a **new container**
*   **Recursively walks every nested object**
*   Creates *new objects* for all mutable children
*   Handles circular references

```python
deep = copy.deepcopy(big_dict)
```

### Performance implications

❌ Many allocations  
❌ Recursive traversal  
❌ Duplicate data in memory

👉 Necessary only when **full isolation** is required

***

## Example: Performance Intuition

### Data Structure

```python
data = {
    "ids": list(range(1_000_000)),
    "meta": {
        "source": "system",
        "valid": True
    }
}
```

### What happens

*   **Shallow copy**
    *   Copies `data`
    *   `ids` list is **shared**
    *   `meta` dict is **shared**

*   **Deep copy**
    *   Copies `data`
    *   Copies entire 1M-element list
    *   Copies nested dictionary

### Practical Result

*   Shallow copy: **milliseconds**
*   Deep copy: **hundreds of milliseconds or more**
*   Memory: deep copy duplicates \~1M integers

***

## Microbenchmark-Style Illustration

```python
import copy, time

start = time.perf_counter()
copy.copy(data)
print("Shallow:", time.perf_counter() - start)

start = time.perf_counter()
copy.deepcopy(data)
print("Deep:", time.perf_counter() - start)
```

Typical outcome:

    Shallow: ~0.0001s
    Deep:    ~0.2s+

(The exact numbers depend on system and structure depth, but the **order-of-magnitude difference** is consistent.)

***

## Space (Memory) Cost Comparison

### Shallow Copy

    Original
     ├── list ─────┐
     └── dict ─────┘
    Shared references

✔ One new container  
✔ Same nested objects

### Deep Copy

    Original        Deep Copy
     ├── list  →   ├── list (new)
     └── dict  →   └── dict (new)

❌ Full duplication  
❌ High memory pressure (can trigger GC overhead)

***

## When Performance Really Matters

### ✅ Prefer Shallow Copy When

*   Data is **read-mostly**
*   Nested objects are **immutable**
*   Performance or memory is critical
*   Multithreaded read-only structures

### ✅ Prefer Deep Copy When

*   You **will modify nested data**
*   Isolation is required (e.g., simulations, undo states)
*   Avoiding side effects > performance

***

## Advanced Tip: Selective Deep Copy (Best of Both)

Instead of deep-copying everything:

```python
new_data = data.copy()
new_data["meta"] = data["meta"].copy()  # copy only what you mutate
```

✅ Faster than `deepcopy()`  
✅ Safer than shallow copy  
✅ Common in **production-grade Python**

***

## Interview-Ready Summary

> **Shallow copy is O(n), fast, and memory-efficient but shares nested objects.  
> Deep copy is O(N), slower, memory-heavy, but fully independent.**

**Selective deep copy** means:  
👉 *You shallow‑copy the outer structure, and deep‑copy **only the parts you plan to modify***.

This gives you **most of the safety of `deepcopy()`** with **much better performance**.

***

## Why Selective Deep Copy Exists

`copy.deepcopy()`:

*   Copies **everything**
*   Slow & memory‑heavy for large structures

Shallow copy:

*   Very fast
*   But unsafe if you mutate nested objects

✅ **Selective deep copy is the middle ground**

***

## Basic Pattern

```python
new_obj = original.copy()        # shallow copy
new_obj["nested"] = original["nested"].copy()  # deep copy only this part
```

***

## Example 1: Dictionary with Mixed Data

### Original data

```python
data = {
    "user": "Alice",                     # immutable
    "roles": ["admin", "editor"],        # mutable
    "settings": {
        "theme": "dark",
        "notifications": True
    },                                   # mutable
    "cache": None                        # shared is OK
}
```

We want to:

*   ✅ Modify `roles`
*   ✅ Modify `settings`
*   ❌ Leave everything else shared

***

### Selective Deep Copy Implementation

```python
new_data = data.copy()                     # shallow copy outer dict
new_data["roles"] = data["roles"].copy()   # deep copy list
new_data["settings"] = data["settings"].copy()  # deep copy dict
```

***

### Modify safely

```python
new_data["roles"].append("viewer")
new_data["settings"]["theme"] = "light"
```

***

### Result

```python
print(data)
```

```text
{
 'user': 'Alice',
 'roles': ['admin', 'editor'],
 'settings': {'theme': 'dark', 'notifications': True},
 'cache': None
}
```

```python
print(new_data)
```

```text
{
 'user': 'Alice',
 'roles': ['admin', 'editor', 'viewer'],
 'settings': {'theme': 'light', 'notifications': True},
 'cache': None
}
```

✅ No unintended side effects  
✅ No expensive full deepcopy

***

## Example 2: Nested Structures (One Level Deep)

```python
data = {
    "config": {
        "db": {"host": "localhost", "port": 5432},
        "retry": 3
    },
    "metrics": [1, 2, 3]
}
```

Suppose:

*   You only modify `metrics`
*   Not touching `config`

### Selective deep copy

```python
new_data = data.copy()
new_data["metrics"] = data["metrics"].copy()
```

✅ `config` stays shared  
✅ `metrics` is independent

***

## Example 3: Avoiding `deepcopy()` in High‑Performance Code

### ❌ Bad (expensive)

```python
import copy
new_state = copy.deepcopy(state)
```

### ✅ Better (targeted)

```python
new_state = state.copy()
new_state["position"] = state["position"][:]   # list
new_state["metadata"] = state["metadata"].copy()  # dict
```

This pattern is common in:

*   Game loops
*   Data pipelines
*   State transitions
*   APIs

***

## Example 4: With Custom Objects

```python
class Config:
    def __init__(self, flags):
        self.flags = flags

config = Config(flags={"debug": True})

new_config = Config(config.flags.copy())
new_config.flags["debug"] = False
```

✅ Only the mutable attribute is copied  
✅ Object stays lightweight

***

## Memory Reference Proof

```python
print(data["roles"] is new_data["roles"])       # False
print(data["settings"] is new_data["settings"]) # False
print(data["cache"] is new_data["cache"])       # True
```

This proves **precise control** over what is copied.

***

## When to Use Selective Deep Copy ✅

Use it when:

*   You **know exactly which fields will mutate**
*   Performance or memory matters
*   Objects are large or deeply nested

Avoid it when:

*   Many nested mutations occur unpredictably
*   Complex graphs or circular references exist → use `deepcopy`
