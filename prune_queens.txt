# ✅ What Is Pruning in N‑Queens?

**Pruning** means **cutting off** (not exploring) branches of the recursion tree that **cannot possibly lead to a valid solution**.

👉 Instead of exploring *all* board configurations, we **stop early** when a partial placement already violates constraints.

In N‑Queens, pruning is the **core reason** backtracking works efficiently.

***

## 1. Why Pruning Is Needed

### Without Pruning (Brute Force)

*   Try N choices in every row
*   Total possibilities ≈ **Nⁿ**
*   Extremely slow

### With Pruning

*   As soon as a queen conflicts, we **stop exploring that path**
*   Search space reduces toward **O(N!)**

✅ Pruning makes the problem feasible even for larger N.

***

## 2. What Exactly Do We Prune in N‑Queens?

When trying to place a queen at position:

    (row, col)

We **prune** (skip further recursion) **if ANY of these fail**:

***

### 🚫 1. Column Conflict Pruning

If another queen already occupies the column:

    Q
    |
    |
    X ← prune here

```python
if col in cols:
    continue  # PRUNE
```

✅ Because two queens in same column can attack.

***

### 🚫 2. Left Diagonal Conflict Pruning (↖)

Cells where:

    row - col = constant

Example:

    Q
      \
        X ← prune

```python
if (row - col) in diag1:
    continue  # PRUNE
```

***

### 🚫 3. Right Diagonal Conflict Pruning (↗)

Cells where:

    row + col = constant

Example:

        Q
      /
    X ← prune

```python
if (row + col) in diag2:
    continue  # PRUNE
```

***

## 3. How Pruning Works Step‑by‑Step (Example N = 4)

Assume we placed queens at:

    Row 0 → column 1
    Row 1 → column 3

Board looks like:

    . Q . .
    . . . Q
    . . . .
    . . . .

***

### Now at Row 2 – Try all columns

| Column | Reason                      |
| ------ | --------------------------- |
| 0      | ✅ allowed                   |
| 1      | ❌ column conflict → PRUNE   |
| 2      | ❌ diagonal conflict → PRUNE |
| 3      | ❌ column conflict → PRUNE   |

✅ Only column 0 is explored  
❌ Other branches are **cut immediately**

**That’s pruning in action.**

***

## 4. Where Pruning Happens in Code

This line is the **pruning gate**:

```python
if col in cols or (row - col) in diag1 or (row + col) in diag2:
    continue   # PRUNE HERE
```

👉 `continue` means:

*   Do **not** go deeper
*   Skip recursion call
*   Try next column

***

## 5. Pruning vs Backtracking (Important Distinction)

| Concept      | Meaning                               |
| ------------ | ------------------------------------- |
| Backtracking | Undoing a choice after exploring      |
| Pruning      | **Not exploring** a bad choice at all |

✅ Pruning happens **before recursion**  
✅ Backtracking happens **after recursion**

***

## 6. Visual Recursion Tree (With Pruning)

Without pruning:

    Row 0
     ├─ Col 0
     │   ├─ Col 0 ❌
     │   ├─ Col 1 ❌
     │   ├─ Col 2
     │   └─ Col 3
     ├─ Col 1
     │   ├─ ...

With pruning:

    Row 0
     ├─ Col 0
     │   └─ Col 2
     ├─ Col 1
     │   └─ Col 3

🎯 Huge parts of the tree are **never explored**

***

## 7. Why Sets Enable Fast Pruning

Using sets:

```python
cols
diag1
diag2
```

✅ Conflict checks in **O(1)**  
✅ Faster pruning  
✅ Efficient solution

Without sets → scanning board → slow pruning

***

## 8. Impact of Pruning on Complexity

| Aspect          | Without Pruning | With Pruning |
| --------------- | --------------- | ------------ |
| States explored | Nⁿ              | \~N!         |
| Conflict check  | Slow            | O(1)         |
| Scalability     | Poor            | Good         |

***

## 9. How to Explain Pruning in an Interview (Perfect Answer)

> “Pruning means we stop exploring a path as soon as it violates constraints.  
> In N‑Queens, before placing a queen, we check column and diagonal conflicts.  
> If a conflict exists, we skip that choice and avoid unnecessary recursion.”

✅ This answer scores well.

***

## 10. One‑Line Definition (Memorize ✅)

> **Pruning in N‑Queens means skipping recursive paths that already violate column or diagonal constraints.**
