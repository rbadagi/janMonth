Below is a **clear, beginner‑to‑interview level explanation of a recursion tree**, using **N‑Queens (N = 4)** as the example, because that’s where recursion trees + pruning really make sense.

***

# ✅ What Is a Recursion Tree?

A **recursion tree** is a **visual way to understand recursive calls**.

*   Each **node** = one recursive call
*   Each **level** = one step deeper (one row in N‑Queens)
*   Each **branch** = one choice tried
*   **Cut branches** = pruned paths (invalid choices)

***

## 1. Recursion Tree for N‑Queens (High Level)

For **N = 4**, we place queens **row by row**.

*   Level 0 → row 0
*   Level 1 → row 1
*   Level 2 → row 2
*   Level 3 → row 3

Each level represents **one recursive call**.

***

## 2. Root of the Tree (Row 0)

At row 0, we can try all columns:

    Row 0
     ├─ Col 0
     ├─ Col 1
     ├─ Col 2
     └─ Col 3

Each branch means:

> “Place queen at (0, col) and recurse”

***

## 3. Expand One Branch (Row 0 → Col 0)

Now we go to **row 1**.

Some choices are invalid due to **column/diagonal conflicts**.

    Row 0: Col 0
       │
       └── Row 1
            ├─ Col 0 ❌ (same column → pruned)
            ├─ Col 1 ❌ (diagonal → pruned)
            ├─ Col 2 ✅
            └─ Col 3 ✅

✅ Only columns 2 and 3 continue  
❌ Others are pruned immediately

***

## 4. Expand One Valid Path Further

Take `(row 1, col 2)`:

    Row 0: Col 0
       │
    Row 1: Col 2
       │
    Row 2
     ├─ Col 0 ❌ (column conflict)
     ├─ Col 1 ❌ (diagonal conflict)
     ├─ Col 2 ❌ (column conflict)
     └─ Col 3 ❌ (diagonal conflict)

🚫 No valid move at row 2  
👉 **Dead end → backtrack**

This entire subtree **dies here**.

***

## 5. Try the Next Branch

Now switch to `(row 1, col 3)`:

    Row 0: Col 0
       │
    Row 1: Col 3
       │
    Row 2
     ├─ Col 0 ✅
     ├─ Col 1 ❌
     ├─ Col 2 ❌
     └─ Col 3 ❌

Continue recursion:

    Row 3
     ├─ Col 0 ❌
     ├─ Col 1 ❌
     ├─ Col 2 ✅ ✅ ✅ (SOLUTION FOUND)
     └─ Col 3 ❌

🎉 One valid solution found!

***

## 6. Full (Simplified) Recursion Tree

    Row 0
    ├─ Col 0
    │   ├─ Col 2 → ❌
    │   └─ Col 3
    │       └─ Col 0
    │           └─ Col 2 ✅
    ├─ Col 1
    │   └─ Col 3
    │       └─ Col 0
    │           └─ Col 2 ✅
    ├─ Col 2 → similar checks
    └─ Col 3 → similar checks

✅ Valid solutions are **leaf nodes**
❌ Pruned branches stop early

***

## 7. How Pruning Appears in the Tree

Pruning means **branches stop growing**:

    Row 2
     ├─ Col 0 ❌
     ├─ Col 1 ❌
     ├─ Col 2 ❌
     └─ Col 3 ❌   ← STOP HERE

No recursive call below this level.

***

## 8. Why Recursion Tree Is Important (Interview Angle)

Interviewers want to see:

*   Do you understand **how recursion expands**
*   Where **branches are cut**
*   Why complexity reduces

Good explanation:

> “Each node is a recursive call. We try columns row by row.  
> When a conflict occurs, that branch of the recursion tree is pruned.”

✅ That’s a strong answer.

***

## 9. Relation to Time Complexity

*   Without pruning → **Nⁿ nodes**
*   With pruning → roughly **N! nodes**
*   Recursion tree shows **why exponential paths reduce**

***

## 10. Key Takeaways (Memorize ✅)

*   ✅ Levels = rows
*   ✅ Branches = column choices
*   ✅ Leaves = valid solutions
*   ✅ Cut branches = pruning
*   ✅ Backtracking = moving up the tree

Below is a **clear, interview‑ready explanation of recursion tree depth**, tied directly to **N‑Queens** and recursion in general.

***

# ✅ What Is Recursion Tree Depth?

**Recursion tree depth** is the **maximum number of recursive calls made along any one path** from the root (start) to a leaf (end) in the recursion tree.

👉 In simple words:

> **How deep the recursion goes before it stops**

***

## 1. Depth vs Height (Simple Definition)

*   **Depth** = number of levels in the recursion tree
*   **Height of tree** = same as maximum depth

If the deepest recursive call stack has **k calls**, then:

    Recursion Tree Depth = k

***

## 2. Recursion Tree Depth in N‑Queens

### Key Observation

In N‑Queens:

*   Each recursive call places **one queen in one row**
*   Recursion moves **row by row**

So:

    Row 0 → Row 1 → Row 2 → ... → Row (N-1)

✅ Therefore:

> **Maximum recursion depth = N**

***

### Example: N = 4

    Level 0 → backtrack(0)  # row 0
    Level 1 → backtrack(1)  # row 1
    Level 2 → backtrack(2)  # row 2
    Level 3 → backtrack(3)  # row 3
    Level 4 → backtrack(4)  # base case

✅ **Recursion depth = 4 (or 5 including base case)**

***

## 3. Visual Representation

    Depth 0: Row 0
             |
    Depth 1: Row 1
             |
    Depth 2: Row 2
             |
    Depth 3: Row 3
             |
    Depth 4: Solution / Backtrack

📌 Each vertical step = **one recursive call deeper**

***

## 4. Why Recursion Tree Depth Matters

### 1️⃣ Space Complexity

Each recursive call uses stack memory.

    Space Complexity = O(depth)

For N‑Queens:

    Space = O(N)

***

### 2️⃣ Stack Overflow Risk

*   Large depth → more stack usage
*   Languages with small stack limits may crash

***

### 3️⃣ Independent of Pruning

Important interview point:

> **Pruning reduces number of nodes, NOT depth**

*   Width ↓ (fewer branches)
*   Depth stays = N

***

## 5. Depth vs Number of Nodes (Very Important)

| Concept | Meaning             | N‑Queens |
| ------- | ------------------- | -------- |
| Depth   | Max recursion stack | N        |
| Nodes   | Total calls made    | \~N!     |
| Breadth | Choices per level   | ≤ N      |

✅ Interviewers often confuse candidates with this

***

## 6. General Formula for Recursion Depth

### Linear Recursion

```python
f(n):
    f(n - 1)
```

📌 Depth = **O(n)**

***

### Divide & Conquer (Binary recursion)

```python
f(n):
    f(n/2)
```

📌 Depth = **O(log n)**

***

### Multiple Calls (Backtracking)

```python
f(level):
    for choice in choices:
        f(level + 1)
```

📌 Depth = **number of levels**  
📌 Nodes = exponential

***

## 7. Interview Question Examples

### Q1: What is recursion depth in N‑Queens?

✅ **Answer:** O(N), because we place one queen per row.

***

### Q2: Can pruning reduce recursion depth?

✅ **Answer:** No. Pruning reduces explored nodes, not depth.

***

### Q3: What causes stack overflow in recursion?

✅ **Answer:** Excessive recursion depth or missing base case.

***

## 8. Common Mistakes to Avoid

❌ Saying depth is O(N!)  
❌ Confusing depth with total calls  
❌ Ignoring base case level  
❌ Linking pruning with depth reduction

***

## 9. One‑Line Interview Definition ✅

> **Recursion tree depth is the maximum number of recursive calls active at any time along a single path.**

***

## 10. Final Summary (Memorize This)

*   ✅ Depth = max stack size
*   ✅ In N‑Queens: **depth = N**
*   ✅ Space complexity depends on depth
*   ✅ Pruning affects width, not depth
