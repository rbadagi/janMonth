## 1️⃣ Basic Level Order Traversal

**Problem:** Print tree nodes level by level.

### Example

        1
       / \
      2   3

Output: `[[1], [2, 3]]`

### Code

```python
from collections import deque

def levelOrder(root):
    if not root:
        return []

    result = []
    q = deque([root])

    while q:
        level = []
        size = len(q)

        for _ in range(size):
            node = q.popleft()
            level.append(node.val)

            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)

        result.append(level)

    return result
```

## 3️⃣ Reverse Level Order Traversal

**Problem:** Print bottom levels first.

    Output: [[4,5,6,7], [2,3], [1]]

```python
def reverseLevelOrder(root):
    if not root:
        return []

    result = []
    q = deque([root])

    while q:
        level = []
        for _ in range(len(q)):
            node = q.popleft()
            level.append(node.val)

            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)

        result.insert(0, level)

    return result
```

## 4️⃣ Zigzag (Spiral) Level Order Traversal

**Problem:** Alternate left → right, right → left.

    Output: [[1], [3,2], [4,5,6,7]]

```python
def zigzagLevelOrder(root):
    if not root:
        return []

    result = []
    q = deque([root])
    leftToRight = True

    while q:
        level = deque()
        for _ in range(len(q)):
            node = q.popleft()

            if leftToRight:
                level.append(node.val)
            else:
                level.appendleft(node.val)

            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)

        result.append(list(level))
        leftToRight = not leftToRight

    return result
```

## 5️⃣ Maximum Element at Each Level

**Problem:** Find max value at every tree level.

```python
def maxLevelValues(root):
    if not root:
        return []

    result = []
    q = deque([root])

    while q:
        max_val = float('-inf')
        for _ in range(len(q)):
            node = q.popleft()
            max_val = max(max_val, node.val)

            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)

        result.append(max_val)

    return result
```

## 6️⃣ Average of Levels in Binary Tree

```python
def averageOfLevels(root):
    if not root:
        return []

    result = []
    q = deque([root])

    while q:
        level_sum = 0
        size = len(q)

        for _ in range(size):
            node = q.popleft()
            level_sum += node.val

            if node.left:
                q.append(node.left)
            if node.right:
                q.append(node.right)

        result.append(level_sum / size)

    return result
```

## 7️⃣ Minimum Depth of Binary Tree (Using BFS)

**Why BFS?** Stops as soon as first leaf is found ✅

```python
def minDepth(root):
    if not root:
        return 0

    q = deque([(root, 1)])

    while q:
        node, depth = q.popleft()

        if not node.left and not node.right:
            return depth

        if node.left:
            q.append((node.left, depth + 1))
        if node.right:
            q.append((node.right, depth + 1))
```

# ✅ BFS vs DFS — Key Differences Explained

## 1️⃣ Basic Idea

| Algorithm | Full Form            | Strategy                             |
| --------- | -------------------- | ------------------------------------ |
| **BFS**   | Breadth‑First Search | Explores **level by level**          |
| **DFS**   | Depth‑First Search   | Goes **deep first**, then backtracks |

***

## 2️⃣ Traversal Style (Most Important Difference)

### 🔹 BFS (Level‑wise)

    Start → All neighbors → Next level → Next level

Uses a **queue**.

### 🔹 DFS (Depth‑wise)

    Start → Go deep → Backtrack → Explore siblings

Uses **recursion or stack**.

***

## 3️⃣ Data Structure Used

| Algorithm | Data Structure    |
| --------- | ----------------- |
| **BFS**   | Queue             |
| **DFS**   | Stack / Recursion |

***

## 4️⃣ Pseudocode Comparison

### ✅ BFS

```python
queue = [start]
visited = set([start])

while queue:
    node = queue.pop(0)
    for neighbor in node.neighbors:
        if neighbor not in visited:
            visited.add(neighbor)
            queue.append(neighbor)
```

### ✅ DFS

```python
def dfs(node):
    visited.add(node)
    for neighbor in node.neighbors:
        if neighbor not in visited:
            dfs(neighbor)
```

***

## 5️⃣ Time Complexity (Same for Both)

| Algorithm | Time Complexity |
| --------- | --------------- |
| BFS       | **O(V + E)**    |
| DFS       | **O(V + E)**    |

`V = vertices`, `E = edges`

✅ Interview trick question — **time is same**, space differs.

***

## 6️⃣ Space Complexity (Important!)

| Algorithm | Space Usage                 |
| --------- | --------------------------- |
| **BFS**   | Higher (stores whole level) |
| **DFS**   | Lower (stores path only)    |

✅ BFS may use more memory on **wide graphs**  
✅ DFS may overflow stack on **deep graphs**

***

## 7️⃣ Shortest Path Guarantee ⭐

| Algorithm | Shortest Path?           |
| --------- | ------------------------ |
| **BFS**   | ✅ YES (unweighted graph) |
| **DFS**   | ❌ NO                     |

📌 **Interview favorite line:**

> *“BFS guarantees the shortest path in an unweighted graph because it explores nodes in increasing distance order.”*

***

## 8️⃣ When to Use BFS vs DFS (Very Important!)

### ✅ Use BFS When:

*   Shortest path needed
*   Minimum depth
*   Level‑wise traversal
*   Unweighted graph

📌 Examples:

*   Word Ladder
*   Rotting Oranges
*   Binary Tree Level Order
*   Shortest path in matrix

***

### ✅ Use DFS When:

*   Exploring all paths
*   Backtracking problems
*   Cycle detection
*   Topological sorting

📌 Examples:

*   Maze solving
*   Sudoku
*   Permutations
*   Topological sort

***

## 9️⃣ Real‑World Analogies (Interviewers Love This)

### 🔹 BFS

> Searching nearby rooms first during rescue  
> Finding closest hospital  
> Social network “friends of friends”

### 🔹 DFS

> Exploring a cave deeply  
> Solving a maze  
> Folder/file system traversal

***

## 🔑 One‑Line Interview Answers

✅ **Difference in one line:**

> BFS explores neighbors first, DFS explores depth first.

✅ **Why BFS for shortest path?**

> Because BFS explores nodes level by level, ensuring minimum edges.

✅ **Why DFS is faster sometimes?**

> Less memory usage for deep structures.

# 🚨 Most Tricky BFS–DFS Interview Traps (Must‑Know)

## 1️⃣ Choosing DFS When BFS Is Required (Shortest Path Trap)

### ❌ Common Mistake

Using **DFS** for shortest path in an **unweighted** graph.

```text
Many candidates say:
“I’ll use DFS and track minimum length”
```

### ✅ Correct Insight

> **BFS ALWAYS guarantees shortest path in unweighted graphs**

✅ Because BFS explores nodes **level by level**

📌 Examples:

*   Word Ladder
*   Shortest path in matrix
*   Rotting Oranges
*   Nearest exit in maze

**Interview Red Flag:**  
Using DFS + global min variable ❌

***

## 2️⃣ Forgetting to Mark `visited` at the Right Time

### ❌ Buggy BFS

```python
if neighbor not in visited:
    queue.append(neighbor)
# visited added later ❌
```

### ✅ Correct BFS

```python
visited.add(neighbor)
queue.append(neighbor)
```

### ⚠️ Why This Is Tricky

*   Same node can be added multiple times
*   Causes **TLE or infinite loop**

📌 Interviewers LOVE asking:

> “When exactly do you mark visited?”

✅ **Answer:**

> Immediately when adding to queue/stack

***

## 3️⃣ Using BFS but Forgetting Level Tracking

### ❌ Mistake

Using BFS but **not counting levels** → wrong distance/output.

### ✅ Pattern

```python
while queue:
    size = len(queue)
    for _ in range(size):
        ...
    level += 1
```

📌 Asked in:

*   Minimum depth of tree
*   Time to rot oranges
*   Distance problems

***

## 4️⃣ Stack Overflow with DFS (Depth Trap)

### ❌ Mistake

Using recursive DFS on very deep inputs.

📌 Example:

*   Long linked list
*   Deep skewed tree

### ✅ Safe Answer

> “DFS recursion may cause stack overflow; iterative DFS or BFS is safer.”

✅ Interview brownie points if you say this.

***

## 5️⃣ Thinking BFS Uses Less Memory than DFS ❌

### 🚫 Wrong Belief

> “BFS uses less memory”

### ✅ Reality

| Algorithm | Memory Risk          |
| --------- | -------------------- |
| BFS       | ❌ Stores whole level |
| DFS       | ✅ Stores path only   |

📌 BFS explodes on **wide graphs**  
📌 DFS explodes on **deep graphs**

***

## 6️⃣ Using DFS Instead of BFS in Grid Distance Problems

### ❌ Wrong Approach

DFS from each cell → O(N³)

### ✅ Correct Insight

**Multi‑source BFS**

📌 Classic questions:

*   01 Matrix
*   Rotting Oranges
*   Nearest gate
*   Distance to nearest zero

✅ Start BFS from **all sources at once**

***

## 7️⃣ Confusing Tree BFS vs Graph BFS

### ❌ Mistake

Skipping `visited` in graph BFS because it worked in trees.

### ✅ Why This Fails

Graphs can have:

*   Cycles
*   Multiple paths

✅ Trees don’t need visited  
✅ Graphs ALWAYS need visited

***

## 8️⃣ Saying DFS Is Faster Than BFS (Trap Question)

### 🚫 Incorrect

> “DFS is faster”

### ✅ Correct Answer

> “Both have time complexity O(V + E)”

📌 Interview follow‑up:

> Then why use one over the other?

✅ Answer with **use cases**, not speed.

***

## 9️⃣ Forgetting Parent Tracking (Path Reconstruction)

### ❌ Mistake

Finding destination but can’t show path.

### ✅ Correct Pattern

```python
parent[child] = node
```

📌 Asked in:

*   Shortest path print
*   Maze problems
*   Graph traversal output

***

## 🔟 Confusing BFS vs DFS in Cycle Detection

### ✅ Undirected Graph

*   DFS → track parent
*   BFS → track parent

### ✅ Directed Graph

*   DFS → recursion stack
*   BFS → indegree (Kahn’s algorithm)

📌 Interview Trap:  
Using BFS visited logic for directed cycle ❌

***

## 1️⃣1️⃣ Wrong Order of Operations (Subtle Bug)

### ❌ Wrong

```python
pop → process → mark visited
```

### ✅ Correct

```python
mark visited → push
```

This causes:

*   Duplicate visits
*   Infinite loops

***

## 1️⃣2️⃣ Forgetting BFS Can Early‑Exit

### ❌ Inefficient

Searching full graph even after finding answer.

### ✅ Smart BFS

> As soon as target is found → stop

📌 Especially for:

*   Minimum steps
*   Nearest value problems
