# ✅ What Is DFS (Depth‑First Search)?

**DFS** is a graph/tree traversal technique where we:

> **Go as deep as possible along one path before backtracking and exploring other paths**

It follows the idea:
**Explore → Go deeper → Hit dead end → Backtrack**

DFS is commonly implemented using:

*   **Recursion** (implicit stack), or
*   **Explicit stack**

***

# ✅ Working Graph Example

## Graph (Undirected)

            A
           / \
          B   C
         / \   \
        D   E   F
             \
              G

### Adjacency List Representation

```python
graph = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [],
    'E': ['G'],
    'F': [],
    'G': []
}
```

***

# ✅ DFS Traversal Goal

Start DFS from node **A**  
👉 Visit nodes using **depth‑first order**

***

# ✅ DFS Step‑by‑Step Walkthrough (Recursive)

### Step 1: Start at A

    Visit A
    Go to B

### Step 2: At B

    Visit B
    Go to D

### Step 3: At D

    Visit D
    No neighbors → backtrack to B

### Step 4: Back at B

    Go to E

### Step 5: At E

    Visit E
    Go to G

### Step 6: At G

    Visit G
    Dead end → backtrack to E → backtrack to B → backtrack to A

### Step 7: Back at A

    Go to C

### Step 8: At C

    Visit C
    Go to F

### Step 9: At F

    Visit F
    Dead end → traversal complete

***

# ✅ DFS Traversal Order (Final Output)

    A → B → D → E → G → C → F

✅ You can see DFS **fully explores one branch before moving to another**

***

# ✅ Working DFS Code (Python – Recursive)

```python
def dfs(graph, node, visited):
    if node not in visited:
        print(node, end=" ")
        visited.add(node)
        for neighbor in graphdfs(graph, neighbor, visited)

graph = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [],
    'E': ['G'],
    'F': [],
    'G': []
}

visited = set()
dfs(graph, 'A', visited)
```

### ✅ Output

    A B D E G C F

***

# ✅ DFS Using Stack (Iterative – Also Working)

```python
def dfs_iterative(graph, start):
    visited = set()
    stack = [start]

    while stack:
        node = stack.pop()
        if node not in visited:
            print(node, end=" ")
            visited.add(node)
            # reverse for same order as recursive
            for neighbor in reversed(graph[node]):
                stack.append(neighbor)

dfs_iterative(graph, 'A')
```

✅ Output:

    A B D E G C F

# ✅ DFS Explained Using the Recursion Stack

## 1. What Is the Recursion Stack?

When a function calls itself (recursion), **each function call is pushed onto a stack** called the **recursion stack**.

*   The **top of the stack** is the currently executing call
*   When a call finishes, it is **popped** from the stack
*   DFS relies on this stack to remember **where to return**

👉 **DFS = recursion + recursion stack**

***

## 2. Tree Example (We’ll Use This)

            A
           / \
          B   C
         / \
        D   E

We’ll perform **DFS (Preorder)**:

    Root → Left → Right

Expected traversal:

    A → B → D → E → C

***

## 3. DFS Code (Simple)

```python
def dfs(node):
    if node is None:
        return

    print(node.val)
    dfs(node.left)
    dfs(node.right)
```

***

## 4. How the Recursion Stack Works (Step by Step)

### 🔹 Step 1: Start DFS at A

Call:

```python
dfs(A)
```

**Recursion Stack:**

    dfs(A)

Output:

    A

***

### 🔹 Step 2: DFS goes to left child B

Call:

```python
dfs(B)
```

**Recursion Stack:**

    dfs(A)
      └── dfs(B)

Output so far:

    A B

***

### 🔹 Step 3: DFS goes to left child D

Call:

```python
dfs(D)
```

**Recursion Stack:**

    dfs(A)
      └── dfs(B)
            └── dfs(D)

Output:

    A B D

***

### 🔹 Step 4: D has no children → return (POP)

`dfs(D)` finishes.

**Recursion Stack after pop:**

    dfs(A)
      └── dfs(B)

DFS now continues to **B’s right child (E)**.

***

### 🔹 Step 5: DFS goes to E

Call:

```python
dfs(E)
```

**Recursion Stack:**

    dfs(A)
      └── dfs(B)
            └── dfs(E)

Output:

    A B D E

***

### 🔹 Step 6: E finishes → backtrack to B → back to A

**Popping happens automatically:**

    dfs(E) → pop
    dfs(B) → pop

**Stack now:**

    dfs(A)

DFS now moves to **A’s right child (C)**.

***

### 🔹 Step 7: DFS goes to C

Call:

```python
dfs(C)
```

**Recursion Stack:**

    dfs(A)
      └── dfs(C)

Output:

    A B D E C

***

### 🔹 Step 8: C finishes → A finishes → Stack empty

DFS traversal complete ✅

***

## 5. Final Traversal Output

    A → B → D → E → C

***

## 6. Key Insight: Why the Stack Matters

The recursion stack:

*   Remembers **which node to return to**
*   Makes **backtracking automatic**
*   Ensures DFS explores **deep first**

📌 You **don’t write stack code manually** — recursion does it for you.

***

## 7. Visual Summary of Stack Behavior

### Stack Growing (Going Deep)

    dfs(A)
    dfs(B)
    dfs(D)

### Stack Shrinking (Backtracking)

    dfs(D) → pop
    dfs(E)
    dfs(E) → pop
    dfs(B) → pop

***

## 8. DFS Stack Depth (Important for Interviews)

*   **Stack depth = height of the tree**
*   Space complexity:

<!---->

    O(height of tree)

Worst case (skewed tree):

    O(N)

Best case (balanced tree):

    O(log N)

***

## 9. DFS vs Explicit Stack (Same Idea)

| Recursive DFS          | Iterative DFS        |
| ---------------------- | -------------------- |
| Uses call stack        | Uses manual stack    |
| Cleaner code           | More control         |
| Risk of stack overflow | Safer for deep trees |

***

## 10. Interview‑Perfect One‑Liner ✅

> **DFS uses the recursion stack to go deep into a branch, and backtracking happens when function calls return and are popped from the stack.**

***

## 11. Common Mistakes to Avoid

❌ Forgetting base case → infinite recursion  
❌ Confusing recursion stack with BFS queue  
❌ Saying DFS needs extra memory besides stack

***

## 12. Final Takeaways

*   DFS relies on the **recursion stack**
*   Stack tracks the **current path**
*   Popping = backtracking
*   Space complexity depends on **tree height**

# ✅ DFS Using an Iterative Stack (No Recursion)

## 1. Why Iterative DFS?

Recursive DFS uses the **call stack implicitly**.  
**Iterative DFS** does the same thing, but **you manage the stack yourself**.

👉 Same traversal order  
👉 Better control  
👉 Avoids stack overflow for very deep trees/graphs

***

## 2. Tree Example (Same as Before)

            A
           / \
          B   C
         / \
        D   E

We’ll perform **DFS (Preorder)**:

    Root → Left → Right

✅ Expected output:

    A → B → D → E → C

***

## 3. Core Idea of Iterative DFS

1.  Create an empty **stack**
2.  Push the **root**
3.  While stack is not empty:
    *   Pop a node
    *   Visit it
    *   Push its children onto the stack  
        (➡️ **Right child first**, then **Left child**)

📌 Pushing right first ensures left is processed first (LIFO behavior).

***

## 4. Step‑by‑Step Stack Execution

### 🌱 Initial State

    Stack: [A]
    Output: 

***

### 🔹 Step 1: Pop A

*   Visit A
*   Push C, then B

<!---->

    Stack: [C, B]
    Output: A

***

### 🔹 Step 2: Pop B

*   Visit B
*   Push E, then D

<!---->

    Stack: [C, E, D]
    Output: A B

***

### 🔹 Step 3: Pop D

*   Visit D
*   D has no children

<!---->

    Stack: [C, E]
    Output: A B D

***

### 🔹 Step 4: Pop E

*   Visit E
*   No children

<!---->

    Stack: [C]
    Output: A B D E

***

### 🔹 Step 5: Pop C

*   Visit C
*   No children

<!---->

    Stack: []
    Output: A B D E C

✅ Stack empty → DFS complete

***

## 5. Final DFS Traversal Order ✅

    A → B → D → E → C

This exactly matches **recursive DFS (preorder)**.

***

## 6. Working Code (Iterative DFS – Tree)

### Tree Node

```python
class Node:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None
```

### Iterative DFS (Preorder)

```python
def dfs_iterative(root):
    if not root:
        return

    stack = [root]

    while stack:
        node = stack.pop()
        print(node.val, end=" ")

        # Push right first so left is processed first
        if node.right:
            stack.append(node.right)
        if node.left:
            stack.append(node.left)
```

***

## 7. Comparison: Recursive vs Iterative DFS

| Aspect              | Recursive DFS | Iterative DFS   |
| ------------------- | ------------- | --------------- |
| Stack type          | Call stack    | Manual stack    |
| Code                | Cleaner       | More control    |
| Stack overflow risk | Yes           | No (controlled) |
| Traversal order     | Same          | Same            |

✅ Both achieve the same DFS behavior.

***

## 8. Time & Space Complexity

*   **Time Complexity:** `O(N)`  
    (Every node visited once)

*   **Space Complexity:** `O(H)`  
    where `H` = height of tree  
    (Worst case `O(N)` for skewed tree)

***

## 9. Interview‑Perfect One‑Liner ✅

> **Iterative DFS simulates recursive DFS by using an explicit stack, pushing the right child first so that the left child is processed first.**

***

## 10. When to Prefer Iterative DFS

✅ Very deep trees or graphs  
✅ Stack overflow risk with recursion  
✅ Need fine‑grained control over traversal  
✅ Low‑level or system‑critical code

***

## Final Takeaways

*   Iterative DFS = **manual recursion**
*   Stack controls traversal order
*   Push **right before left**
*   Same traversal, safer memory control
